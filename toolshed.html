The Toolshed

An MCP-first platform where the system that builds products is the product. Each component is a Model Context Protocol server — composable, agent-ready, and designed to compound.

The core insight

Most solo builders optimize for shipping products. But products are one-off bets. The real leverage comes from building the system that makes those bets — and making that system smarter with every build.

The Toolshed is that system. It's not a monolithic app. It's a collection of MCP servers that orchestrate the entire build lifecycle: from validating ideas, through AI-assisted development, to post-launch decisions. Each server is independently useful, but together they form a compound asset.

The weekly products are validation experiments. They generate revenue signals and learning data. But the real value accumulates in the Knowledge Base — a structured record of every decision, every pivot, every lesson that makes the next build faster and smarter.

Architecture
Layer 1: Pre-Build Intelligence Validate before building

Most failed builds fail before code starts — wrong problem, wrong market, wrong scope. This layer stress-tests ideas before committing build time.

Layer 2: Build Orchestration AI builds, human validates

AI can generate code, but judgment calls need human oversight. This layer orchestrates the build process with structured checkpoints for human decisions.

Layer 3: Launch & Validate Ship and measure signal

Shipping is the beginning, not the end. This layer tracks what matters after launch and structures the kill/continue/pivot decision.

◆ Human checkpoint: Kill/continue/pivot decisions
The Compound Layer The moat that grows

Every other layer feeds into this one. The Knowledge Base captures what worked, what failed, and why — creating a compound asset that makes every future build better.

Knowledge Base MCP designing Structured learnings from every build, queryable by AI and humans
Why MCP-first

Model Context Protocol is Anthropic's standard for connecting AI models to external tools and data. Each Toolshed component is an MCP server because:

Composable Mix and match components. Use the Decision Framework without the Spec Generator. Swap implementations.
Agent-ready Every component works with Claude, GPT, or any MCP-compatible model. No vendor lock-in.
Inspectable MCP exposes tool calls and context. You can see exactly what the AI is doing and why.
BYOK-friendly Bring Your Own Keys. Users control their API costs, data stays local, trust is earned.
Human non-negotiables

AI orchestrates research, generates code, and surfaces options. But these decisions never get automated:

◆ Go/no-go — Whether to build at all. AI provides evidence; human decides.
◆ Scope lock — What's in v1, what's deferred. AI suggests; human commits.
◆ Architecture calls — Which patterns, which tradeoffs. AI presents options; human chooses.
◆ Design approval — Visual and UX decisions. AI generates; human approves.
◆ Ship decision — Ready or not ready. AI reports status; human calls it.
◆ Kill/continue — Post-launch verdict. AI tracks signals; human decides fate.
Current state
