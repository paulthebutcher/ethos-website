Layer 1: Pre-Build Intelligence planned

Spec Generator

Turns validated ideas into build-ready specifications. Outputs scope doc, tech decisions, and task breakdown.

The problem

The gap between "validated idea" and "build-ready spec" is where most projects go sideways. You know what to build, but the spec is either too vague (leading to scope creep) or too rigid (breaking when reality hits).

Five previous build attempts showed the same pattern: spec quality is the bottleneck. Better specs → faster builds → more learning per week.

The approach

An MCP server that generates structured specs from Decision Framework outputs:

Scope document What's in v1, what's explicitly out, what's deferred to v2.
Tech decisions Stack choices, architecture patterns, key tradeoffs — with reasoning.
Task breakdown Ordered list of build steps, each small enough to complete in one session.
Checkpoint definition Where human decisions are needed during the build.
Output

A spec package that can be handed to Claude (or any AI coding assistant) with minimal back-and-forth:

  • Scope doc with clear boundaries
  • Technical architecture decisions
  • Ordered task list
  • Human checkpoint schedule
  • Success criteria from Decision Framework
