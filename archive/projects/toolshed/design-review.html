Design Review

Visual/UX review workflow with structured feedback capture. Screenshot → feedback → iteration loop.

The problem

AI can generate UI code, but visual judgment is still a human skill. The challenge is creating a fast feedback loop that doesn't become a bottleneck.

Screenshots alone aren't enough. You need structured feedback that the AI can act on — "make it bigger" isn't useful; "increase header font to 24px and add 16px padding" is.

The approach

An MCP server for visual/UX review:

Screenshot capture Automated screenshots at key UI states.
Structured feedback Annotate with specific, actionable changes — not vague impressions.
Iteration tracking Version history of visual changes with decision reasoning.
Pattern library Capture approved patterns for reuse across builds.
Feedback categories

Structured feedback types that AI can act on:

◆ Layout — Spacing, alignment, hierarchy changes.
◆ Typography — Font size, weight, line height adjustments.
◆ Color — Specific color changes with hex/HSL values.
◆ Interaction — Hover states, transitions, feedback patterns.
◆ Approve — Mark as done, capture pattern for reuse.
Status

Current: Planned. Lower priority than core decision/spec flow.

Dependency: Useful once builds reach the UI phase consistently.
