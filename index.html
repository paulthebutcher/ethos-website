What this is

A solo AI product incubator. I build one AI-powered product per week — not to ship fast for its own sake, but because building is the fastest way to learn a problem space. Revenue is the signal; everything else is hypothesis.

The meta-project underneath: a pre-build intelligence pipeline that stress-tests each idea before code starts. Better input → better builds → faster learning.

The loop

02 Research evidence + stress test
03 Scope minimum testable bet
04 Build ship by Friday
05 Measure revenue signal or kill

Operating principles
Revenue is the only reliable signal Build to learn, but measure by whether someone pays. Everything else is theater.
Build the process before the software Run it manually. Automate only what proves valuable. Don't confuse building a tool with improving an outcome.
Risk-first sequencing Identify the thing that might not work. Test it before building everything around it.
Agent-ready by default Every tool should be callable by humans and AI agents from day one. BYOK solves cost, privacy, and trust at once.
Products over personal brand The work is the signal. No content creation for its own sake.
