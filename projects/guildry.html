Guildry

on hold
Started 06 Feb 2026 · guildry.theaiethos.com

Project intelligence for agencies. Scope, staff, deliver, and learn from every project — a system that gets smarter with each project you close.

Thesis

Services firms have years of project data but no way to learn from it. LLMs can now extract patterns from unstructured history and use them to improve future estimates, staffing decisions, and delivery quality.

Architecture

Seven interconnected modules covering the full services business lifecycle. Phase 1 shipped three core modules that form the inner learning loop:

Blueprint
Conversational project scoping with phase suggestions and hour estimates
Bench
Talent network management with skills tracking
Retro
Project retrospectives capturing lessons learned
+ 4 more
Signal, Plinth, Relay, Folio (designed, not built)

Signals

  • Phase 1 complete — Blueprint, Bench, Retro all working
  • Core loop functional: Scope → Staff → Deliver → Learn
  • Built in ~3 hours across 2 sessions — validates rapid development thesis

Technical Decisions

Clerk auth, Supabase with RLS, Claude API integration. AI creates clients via natural language conversation. Pivoted from 17-PR waterfall to discovery-first approach — let AI conversations reveal what data structure needs to be.

Key learning: build guides can be overspecified. Target schema pattern flexible enough to evolve. Vertical slices (one feature end-to-end) work better than horizontal layers.

Status

On hold while focusing on the pre-build intelligence pipeline. Phase 1 is functional and deployed. Remaining 4 modules designed but not built. Would resume if a services firm expressed interest in piloting.
