What This Is

The structured process for taking a shipped MVP from "live URL" to evidence of demand or a kill decision. The Build Program optimizes for learning whether something can be built. The Growth Program optimizes for learning whether anyone cares.

Graduation Criteria

Projects enter Growth after their Friday Build Program retro if the answer to this question is "yes":

"Did anything about this week's build make me think there's a real user on the other side?"

If the answer is "no but I learned something useful," that's a valid Build Program success that doesn't need Growth.

The Four Phases

Phase 1: Exposure

Get the MVP in front of 50-100 people who match the target persona. Small-scale distribution where you can observe behavior. Not a launch—targeted exposure.

Phase 2: Signal Collection

Understand what happened when people encountered the product. Track quantitative metrics (workflow completion, returns) and qualitative feedback (what they said, asked for, got confused by).

Phase 3: Iteration

Make one targeted change based on signals. Not a redesign—one hypothesis about the biggest drop-off point. Time-boxed to 3 hours maximum.

Phase 4: Revenue Test

Learn if anyone will take a payment action. Not "would you pay?" but "here's the price, click buy." Tests willingness to pay, not ability to deliver a paid product yet.

The Verdict

After 4 weeks, one of three outcomes:

Sustain
Revenue signal + returning users + clear path to $500 MRR. Becomes priority product. Gets Build Program weeks for feature development. Rare—maybe 1 in 10.
Orbit
Some interest but not enough to invest heavily. Keep it live, monitor passively, revisit if signals change. Low-maintenance mode. Most "successful" builds.
Kill
No signal. No workflow completions. No revenue action. Document learnings in VERDICT.md and move on. Not a failure—a data point for the Build Program.

Time Budget

Growth Program work cannot compete with Build Program days. Total: ~2-3 hours/week, mostly Monday PM and Friday PM. If a project needs more than 3 hours/week, it should either become the Build Program focus for a week or get killed.

Current Capacity

Start with 1 project at a time. Full attention, clean signal, learn the process. As the process matures, scale to 2-3 with lighter touch per project.

## Pre-Growth Checklist

_Before starting exposure, these must be in place:_

- [ ] Analytics installed (Plausible / Vercel Analytics)
- [ ] Custom event tracking: `workflow_completed` fires when user finishes core action
- [ ] Landing page clearly explains what the product does in < 10 seconds

- [ ] Error states handled (bad input doesn't break the product)

## What NOT To Do

- Don't post in more than 3 channels in week 1. You need to be able to trace which channel drove which signal.
- Don't ask "would you use this?" Ask "try it and tell me where you got stuck."
- Don't mention it's an MVP or that you built it in a week. That frames it as unfinished instead of focused.
- Don't pay for ads yet. If organic distribution can't get 50 eyeballs, the positioning is wrong.

## Revenue Signal Assessment

- [ ] **Strong signal:** Someone paid. Repeat.
- [ ] **Moderate signal:** People clicked buy but didn't complete. Explore why — price? trust? friction?
- [ ] **Mild signal:** Someone asked about pricing unprompted. Interest exists, mechanism is wrong.
- [ ] **No signal:** Nobody engaged with pricing at all. Product isn't solving a problem people value with money.

**Honest read:**
_One paragraph. Is this a product people will pay for? Not "could" — will._

---

## If Revenue Signal is Strong

- [ ] Set up proper billing (if using a hacky test method)
- [ ] Plan next Build Program week focused on this product's paid features
- [ ] Define path to $500 MRR: how many customers at what price?

## If Revenue Signal is Weak

- [ ] Is it a pricing problem? (Would a different price change behavior?)
- [ ] Is it a positioning problem? (Do people understand what they're paying for?)
- [ ] Is it a product problem? (Is the free version not compelling enough to upgrade from?)
- [ ] Is it a market problem? (Nobody will pay for this. Kill it.)
